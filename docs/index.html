Instellar Hermes extends the Hermes-Agent framework with a three-layer security middleware that protects your system. Every critical command is blocked, whitelisted, or sent to your Telegram for approval — before it ever reaches the shell.

Quick Install Source

Three Layers of Protection

Every command passes through a pipeline of security gates. If any layer rejects it, the command never executes.

⛔

Static Filter

Commands containing rm -rf (bare), chmod 777, dd to block devices, or access to /etc/shadow are blocked unconditionally — no exceptions.

🔑

Sudo Whitelist

Only specific sudo commands from your local sudo_whitelist.json are allowed. Any sudo command not in the list is denied immediately.

📱

Telegram Gate

Critical commands (install, mv, write, package managers, etc.) pause execution and send an approval request to your Telegram bot. Approve or deny with one tap.

🛡️

Fail-Safe by Design

If the Telegram bot doesn't respond, the token is missing, or any error occurs — the command is denied. Your system is always protected.

🔌

Non-Intrusive

The middleware integrates seamlessly into terminal_tool() — the existing Hermes guards (tirith, dangerous patterns) still run first. The middleware is an additional layer, not a replacement.

⚡

Easy Setup

Single command install via install.sh. Create your Telegram bot with @BotFather, fill in the config, and you're protected.

Execution Pipeline

Every terminal command flows through this pipeline. The middleware intercepts right before the subprocess call.

# ── Execution flow ──────────────────────────────
LLM → terminal_tool() → Hermes Guards
↓
Security Middleware
├─ Static Filter ⛔ rm -rf / ← blocked
├─ Sudo Whitelist ⛔ sudo rm -rf ← blocked
├─ Telegram Gate ✅ approved → runs
│ ❌ denied → blocked
│ ⏱️ timeout → blocked
↓ ⚠️ error → blocked
subprocess.Popen()
1

Static Filter

Checks command against blocklist patterns. Any match → denied.

2

Sudo Whitelist

If command uses sudo, checks against the whitelist JSON. No match → denied.

3

Telegram Gate

If command is critical, sends to Telegram bot. Waits for ✅ or ❌ — timeout = deny.

4

Execute

Only if all gates pass. Any error in any gate → command never runs.

Quick Install

Get Instellar Hermes running in under a minute. You'll need Python 3.11+, git, and a Telegram bot token.

Clone the repository

This is the Instellar Hermes fork — not the upstream. The middleware is built in.

$ git clone https://github.com/Samuv5/instellar-hermes.git $ cd instellar-hermes

Run the installer

The installer creates a virtual environment, installs dependencies (including python-telegram-bot), and generates the security config template.

$ chmod +x install.sh
$ ./install.sh

Create your Telegram Bot

Open Telegram, search for @BotFather, and send /newbot. Follow the prompts to get your API token. Then send a message to your new bot.

# Find your chat_id — visit this URL in your browser:
https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates

Configure the middleware

Edit ~/.hermes/security_config.json with your bot token, chat ID, and sudo whitelist.

$ nano ~/.hermes/security_config.json

Run Hermes

Activate the virtual environment and start. The middleware is active by default.

$ source ~/.instellar/venv/bin/activate
$ hermes run

Or install manually

# Clone the fork
git clone https://github.com/Samuv5/instellar-hermes.git
cd instellar-hermes

# Create virtual environment
python3 -m venv venv
source venv/bin/activate

# Install with messaging extra (includes python-telegram-bot)
pip install -e .[messaging]

# Create security config
mkdir -p ~/.hermes
cp security_config.example.json ~/.hermes/security_config.json
nano ~/.hermes/security_config.json

# Run
hermes run

Configuration

The middleware reads from ~/.hermes/security_config.json. All fields are optional — set what you need.

Key Type Default Description
enabled bool true Master switch — disable all middleware layers
static_filter bool true Block rm -rf (bare), chmod 777, dd to /dev/, /etc/shadow
sudo_whitelist array [...] List of allowed sudo command prefixes
telegram_gate bool true Enable Telegram approval for critical commands
telegram_token string "" Your bot token from @BotFather
telegram_chat_id string "" Your Telegram chat ID
timeout_seconds int 120 How long to wait for Telegram approval before auto-denying
env_skip array ["docker","singularity","modal","daytona"] Environment types that skip the middleware (containers are isolated)
// ~/.hermes/security_config.json — Full example
{
  "enabled": true,
  "static_filter": true,
  "sudo_whitelist": true,
  "telegram_gate": true,
  "sudo_whitelist": [
    "apt update",
    "apt install",
    "systemctl restart",
    "pip install"
  ],
  "telegram_token": "YOUR_BOT_TOKEN",
  "telegram_chat_id": "YOUR_CHAT_ID",
  "timeout_seconds": 120,
  "env_skip": ["docker", "singularity", "modal", "daytona"]
}
